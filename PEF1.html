1. Higidura Browndarra 2D-n eta 3D-n¶

Higidura browndarrak, eskala makroskopikoan, eskala mikroskopikoko materiaren izaera diskretu edo atomikoaren proba ikusgarrienetako batzuk ematen ditu. Fisikako esparruko prozesu estokastiko ezagunetariko bat da eta, beraz, jariakin batean barreiatzen diren partikula browndarren multzo bat eraikitzea izango da lehenengo helburua.

Notebook honetan, partikulak independenteak direla onartuko dugu, eta baita ere bolumen bera instant berean okupatu dezaketela.

1.1.2. Parametroen def. eta aldagaien hasieratzea¶

# ----------- PARAMETROAK ---------------
dim = 3
dt = 0.05

m = 1.0  
gamma = 1.0 
kBT = 1.0                         # Tenperatura [K], abiadura \sqrt(3) [m/s] izan daiten
sigma = np.sqrt(2*gamma*kBT*dt)   # Desbideratze estandarra (ausazko indarraren modelizazioaren anplituderako)
n_part = 50
n_pausu = 1000

# --------- ALDAGAI HASIERATUAK ----------
R = np.zeros([n_part,dim])             # n partikulen posizioak aldiune batean
V = np.zeros([n_part,dim])             # n partikulen abiadurak aldiune batean
W = np.zeros([n_part,dim])             # n partikulen bulk. met. aldiune batean
R_t = np.zeros([n_pausu,n_part,dim])   # n partikulen posizioen matrizea n_pausu-tarako
V_t = np.zeros([n_pausu,n_part,dim])   # n partikulen abiaduren matrizea n_pausu-tarako
W_t = np.zeros([n_pausu,n_part,dim])   # n partikulen bulk. met.-en matrizea n_pausu-tarako
denbora = np.zeros([n_pausu])

Hemen, simulazioaren parametroak espezifikatzen ditut. 3D-n egingo dut lan, x, y eta z koordenatuekin, eta \(n_p=50\) partikula browndar independenterekin.

  • Simulazio-pausu kopurua 1000-koa da, \(\Delta t = 0.05\) izanik (simulazio-unitatetan).

  • \(\gamma=1.0\) jarri dut, orokortasunik galdu gabe (honi dagokion balio zehatzagoa erantsi diezazkioket geroago). KONTUZ! \(m=k_BT=1.0\) hartu dut. Hauez ez dira, oro har, balio errealistak, baina ez dut orokortasunik galduko.

  • Bigarren atalean, simulaziorako behar izango ditudan array guztiak sortu eta inizializatzen ditut.

  • \(R\): Posizioa. \(V\): Abiadura. \(W\): Auzazko indarra (partikulan eragindakoa). DENBORA-PAUSU AKTUALEAN. \(R_t\), \(V_t\) eta \(W_t\) TRAIEKTORIA OSOARI DAGOZKIEN posizio, abiadura eta auzazko indarra dira.

1.1.3. Simulazioa¶

np.random.seed(0)                           
for i in range(n_pausu):                    # n_pausu-tarako, higidura-ekuazioa ebatzi eta R,V eta W eguneratu  
    W = sigma*np.random.randn(n_part,dim)   # W. prozesua
    V = (1-gamma/m*dt)*V+W/m                # Euler-Maruyama metodoa
    R = R + V*dt
    R_t[i,:,:]=R
    V_t[i,:,:]=V
    W_t[i,:,:]=W
    denbora[i] = i*dt

Kodearen atal nagusi/garrantzitsuena da hauxe. Simulazioa burutzeko, behin da berriz ebazten dugu higidura-ekuazioa partikula jakinaren posizio eta abiadurak eguneratzeko.

Auzazko indarra (partikula eta norabide espazial bakoitzerako) banaketa Gaussiar batek ematen digu (non desbideraketa estandarra sigma den eta “randn” funtzioa erabiliz batazbestekotik, era aleatorioan desbideratzen garen).

Hau da, banaketa Gaussiarrari jarraitzen dion aldagai-sorta bat sor genezake hurrengo eran:

\[x_i = \mu + \sigma*np.random.randn(N)\]

Gure kasuan \(\mu=0\) da eta desbideratze-estandarrak erakutsiko dugu, batazbestean datuak zenbateraino desplaza daitezkeen batazbestekotik.

Aldiuneko posizio eta abiadurak ebatzi eta gero, R_t, V_t eta W_t array-etan gordeko ditugu datu hauek guztiak.

1.2.1. Funtzioak definitu FuncAnimation-erako¶

FuncAnimation: Animazio bat sortzen du, behin eta berriro funtzio bati deituz. Honi buruz jakin behar dudan guztia (parametroak): (https://matplotlib.org/3.1.1/api/_as_gen/matplotlib.animation.FuncAnimation.html -etik hartuta)

  1. fig : matplotlib.figure.Figure

    The figure object that is used to get draw, resize, and any other needed events.

  2. func : callable (animazioa egiteaz arduratuko dena; nik animazio deitu diot)

    The function to call at each frame. The first argument will be the next value in frames. Any additional positional arguments can be supplied via the fargs parameter.

  3. frames : iterable, int, generator function, or None, optional

    Source of data to pass func and each frame of the animation

     - If an iterable, then simply use the values provided. If the iterable has a length, it will override the save_count kwarg.

 - If an integer, then equivalent to passing range(frames)

 - If a generator function, then must have the signature:

  4. init_func : callable, optional (animazioaren hasieratzeaz arduratuko dena; nik hasieratze deitu diot)

    A function used to draw a clear frame. If not given, the results of drawing from the first item in the frames sequence will be used. This function will be called once before the first frame.

  5. interval : number, optional

    Delay between frames in milliseconds. Defaults to 200.

  6. repeat : bool, optional (sekuentzia bukatzean, EZ da animazioa errepikatuko, repeat = False)

    Controls whether the animation should repeat when the sequence of frames is completed. Defaults to True.

  7. blit : bool, optional (nik blit=True)

    Controls whether blitting is used to optimize drawing. Note: when using blitting any animated artists will be drawn according to their zorder. However, they will be drawn on top of any previous artists, regardless of their zorder. Defaults to False.

1.2.2. Hasieratze eta animazio funtzioak (programa nagusian erabiltzeko)¶

def init():
    
    global R, V, W, R_t, V_t, W_t, denbora #aldagai globalak izendatzen ditut, prog. nagusian definitutakoak hemen erabili ahal izateko
    
    ## Aldagaien hasieratzea ##
    R[:,:] = 0.0
    V[:,:] = 0.0
    W[:,:] = 0.0
    R_t[:,:,:] = 0.0
    V_t[:,:,:] = 0.0
    W_t[:,:,:] = 0.0
    denbora[:] = 0.0
    
    ## Objektuen hasieratzea ##
    title.set_text(r'')  # titulu hutsa hasieran
    title2.set_text(r'')
    traiektoria.set_data([],[])          # 2D!!
    traiektoria.set_3d_properties([])    # z ardatza gehitu
    partikulak.set_data([],[])
    partikulak.set_3d_properties([])
    
    return partikulak, title,traiektoria,title2

def animatu(i):
    
    global R, V, W, R_t, V_t, W_t, denbora #aldagai globalak izendatu, prog. nagusian definitutakoak hemen erabili ahal izateko
    
    W = sigma*np.random.randn(n_part,dim)  # ausazko indarren array bat osatuko dut, d_Est: Desbideratze estandarra, n_part: partikula kopurua
    
    R, V = R + V*dt, V*(1-gamma/m*dt)+W/m  # Posizioa eta abiadura eguneratu Euler-Maruyama metodoarekin

    R_t[i,:,:] = R                         # Pausu guztiei dagozkien R, V eta W gorde
    V_t[i,:,:] = V
    W_t[i,:,:] = W 
    denbora[i] = i*dt                     
    
    # Objektuen zehaztapena
    title.set_text(r"t = "+str(denbora[i])+" s.d.u.")  # aldiuneko denbora ikustarazteko  (sdu: simulazioko denbora-unitate)
    title2.set_text(r"OZEANOA")                        # ingurunearen izena
    traiektoria.set_data(R_t[:i+1,n,0],R_t[:i+1,n,1])  # 2D-ko lerroa zehaztu
    traiektoria.set_3d_properties(R_t[:i+1,n,2])       # Honi z osagaia gehitu
    partikulak.set_data(R[:,0],R[:,1])                 # Partikula guztien aldiuneko posizioa zehaztu 2Dn
    partikulak.set_3d_properties(R[:,2])               # Honi z osagaia gehitu
    
    return partikulak,title,traiektoria,title2 # bueltatu FuncAnimation-ek marraztuko dituen objektuak

Partikula Browndar baten higidura simulatu nahi dugu eta honen traiektoria marraztu (horretarako, func_animation funtzioaz baliatuko gara). Horretarako, bi funtzio sortu behar ditut: init(), animazioa hasieratzea ahalbidetuko didana (eta behin bakarrik deituko dena) eta animatu(), aurrerago zehaztuko dudana.

Marraztuko ditugun objektuak (artists) definituko ditut hemen. Objektu hauek global bezala izandatu ditut, programa nagusian erabili ahal izateko. Definitutako eta hasieratutako objektuak:

  1. R: Posizioa.

  2. V: Abiadura.

  3. W: Ausazko indarra.

Bestalde, R_t, V_t eta W_t-k pausu guztiei dagozkien posizio, abiadura eta ausazko indarrak batzen dituzte.

Simulazioan agertuko diren eta marraztuko ditugun beste hiru objektu: titulua, traiektoria eta partikulak. Titulua1 eta Titulua2 -k aldiuneko denbora eta ingurunearen izena adieraziko dute, traiektoriak aukeratutako partikula jakin baten traiektoria eta partikulak objektuak partikula guztiak esferak bailitzan errepresentatuko dizkigu.
